Everything is an API now. The practical skill of being able to stitch two systems together -- read from one, write to another, transform in the middle -- has become one of the most useful things you can do as a technical person.

REST is the dominant pattern: resources have URLs, you GET/POST/PUT/DELETE them, responses come back as JSON. Webhooks are the push version: instead of you polling for changes, the service calls you when something happens. Both are worth being comfortable with.

The annoying parts of working with third-party APIs:

Rate limits: you can only make X requests per minute. Ignore this and you'll get throttled or banned. You need exponential backoff and retry logic.

Auth: OAuth is the standard but every provider implements it slightly differently. API keys are simpler but have their own rotation and security concerns.

Bad documentation: a lot of APIs are underdocumented or have documentation that lags behind the actual behavior. The real spec is often the response you get when you call it, not what the docs say you'll get.

Versioning: APIs change. A field that exists today might be deprecated tomorrow. Defensive parsing (don't assume every field is present) saves you from nasty surprises.

This became the core skill at Nodalli. The product is essentially a set of integrations -- CRMs, email providers, data enrichment APIs, LLM providers -- with orchestration logic and a UI on top. Knowing how to quickly understand and use an unfamiliar API is what makes that kind of product buildable by a small team.
